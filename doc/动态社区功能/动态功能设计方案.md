# 去玩匹配 - 动态社区功能设计方案

## 一、需求分析

### 产品目标
- 增加用户粘性和停留时长
- 通过用户分享内容促进UGC生态建设
- 利用社交互动带来自然裂变增长
- 增加用户互动，提升产品活跃度

### 核心功能
- **两种动态类型**：
  1. **匹配结果动态** - 用户将自己的匹配测试结果分享到社区
  2. **图文动态** - 用户发布个人照片 + 文字的自由动态（新增）
- 其他用户可以浏览、点赞、评论动态
- 支持查看发布者的个人主页和所有动态
- 内容审核机制保障社区安全

### 2.2 页面流程

```
用户流程：
1. 两种发布入口：
   - 完成匹配测试 → 结果页出现"分享到动态社区"按钮 → 直接发布匹配结果动态
   - 动态广场页 → 点击右下角"发布"悬浮按钮 → 进入发布页 → 选择发布图文动态（拍照/从相册选图 + 输入文字）
2. 点击发布 → 内容审核 → 创建动态成功 → 奖励免费次数
3. 在"动态"标签页浏览广场动态 → 两种不同卡片样式展示（匹配结果/图文）→ 点击进入详情
4. 在详情页可以点赞、评论
5. 点击发布者头像 → 进入发布者个人主页 → 查看该用户所有公开动态
6. 在"个人中心" → "我的动态" → 查看/管理自己发布的所有类型动态
7. 可以删除自己的动态
```

### 2.4 功能详情

#### 动态广场 (pages/feeds)
- 下拉刷新最新动态
- 上拉加载更多（分页）
- 按时间倒序排列（最新优先）
- 右下角悬浮"发布"按钮，点击进入发布图文动态页面
- 两种卡片样式：
  - **匹配结果动态**：发布者头像+昵称 → 两人姓名+关系 → 匹配分数 → 简短结论 → 点赞/评论数
  - **图文动态**：发布者头像+昵称 → 文字内容 → 图片网格（1张/多宫格） → 点赞/评论数
- 点击卡片进入详情页
- 点击头像进入发布者个人主页

#### 发布动态页 (pages/feed-publish)
- 支持发布纯文字或图文动态
- 图片选择：从相册选择，最多 9 张
- 文字输入：最多 1000 字
- 选项：是否公开（默认公开）
- 发布按钮：点击后先上传图片，再调用 createFeed 创建动态
- 发布成功后返回广场，奖励免费次数

#### 动态详情页 (pages/feed-detail)
- 根据动态类型不同展示不同内容：
  - **匹配结果动态**：展示完整匹配结果（两人信息、分数、分析、优势、建议）
  - **图文动态**：展示文字内容 + 全屏浏览图片（点击图片可放大预览）
- 展示发布者信息
- 点赞按钮（显示点赞状态）
- 评论区列表展示
- 底部评论输入框
- 支持加载更多评论

#### 发布匹配结果动态（从结果页）
- 在结果页（完整报告解锁后）新增"分享到动态"按钮
- 点击弹出确认框："是否将这个匹配结果发布到动态社区？"
- 选项：是否公开（默认公开）
- 发布成功后提示，并奖励一次免费次数（可配置）
- 只有已解锁的完整结果才能发布

#### 发布图文动态（从动态广场）
- 动态广场右下角悬浮"发布"按钮
- 点击进入独立发布页
- 用户可以：
  - 从相册选择图片（最多9张）
  - 输入文字内容（最多1000字）
  - 选择是否公开
- 发布成功后奖励一次免费次数

#### 点赞功能
- 点击点赞按钮进行点赞/取消点赞
- 实时更新点赞数
- 已点赞按钮高亮显示

#### 评论功能
- 支持用户发表文字评论
- 限制评论长度：1-200字
- 发布者可以删除任何评论，评论者可以删除自己的评论

#### 删除动态
- 只有发布者可以删除自己的动态
- 删除后动态状态变为已删除，不再展示
- 级联删除：删除动态时，相关评论和点赞也标记为删除

#### 我的动态 (pages/my-feeds)
- 展示当前用户发布的所有动态
- 支持删除操作
- 进入个人中心入口

#### 他人主页 (pages/user-home)
- 展示用户信息（昵称、头像）
- 展示该用户发布的所有公开动态列表
- 支持分页加载

### 2.5 产品激励机制
- **发布动态奖励**：每发布一条动态（无论是匹配结果还是图文动态）奖励 1 次免费生成机会（每日最多奖励 2 次）
- **互动本身不消耗次数**：浏览、点赞、评论免费，不限制

## 四、云函数接口设计

### 4.1 createFeed
**入参（匹配结果动态）：**
```javascript
{
  feedType: 1,                 // 1-匹配结果动态
  testRecordId: "string",      // 关联测试记录ID
  isPublic: boolean            // 是否公开，默认true
}
```

**入参（图文动态）：**
```javascript
{
  feedType: 2,                 // 2-图文动态
  content: "string",           // 文字内容
  images: ["string"],          // 图片fileID数组，最多9张
  isPublic: boolean            // 是否公开，默认true
}
```

**出参：**
```javascript
{
  success: true,
  data: {
    feedId: "string",
    reward: 1                  // 奖励次数（0表示今日已达上限）

}
```

## 五、安全与审核

### 5.1 内容安全
- 发布内容（评论）需要调用微信内容安全接口进行文本违规检测
- 在云函数 createFeed 和 addComment 中增加检测
- 检测不通过则禁止发布

### 5.2 权限控制
- 只能删除自己发布的动态
- 只能删除自己发表的评论，或者动态发布者可以删除任何评论
- 私有动态不进入广场列表，仅自己可见
