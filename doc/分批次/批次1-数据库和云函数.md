# OpenChat AI 开发提示词：批次1 - 创建云函数和数据库设计

## 项目背景

「去玩匹配」是一个AI关系匹配测试微信小程序，使用微信原生小程序 + 微信云开发开发。现在需要新增动态社区功能，这是第1批次开发：**只开发云函数部分**，前端页面在下一批次。

## 任务：开发所有新增云函数

请根据设计方案创建以下9个云函数（新增图文动态支持）：

### 需要创建的云函数列表

1. ✅ `createFeed` - 创建动态（支持两种类型：匹配结果/图文）
2. ✅ `listFeeds` - 获取动态列表（广场）
3. ✅ `getFeedDetail` - 获取动态详情
4. ✅ `toggleLike` - 点赞/取消点赞
5. ✅ `addComment` - 添加评论
6. ✅ `listComments` - 获取评论列表
7. ✅ `deleteFeed` - 删除动态
8. ✅ `getUserFeeds` - 获取用户动态列表
9. ✅ `uploadImage` - 图片上传接口（图文动态使用）

## 配置项

奖励机制配置：
- `FEED_REWARD_TIMES = 1` - 每次发布奖励次数
- `FEED_REWARD_MAX_DAILY = 2` - 每日最多奖励次数

发布动态成功后，检查今日发布次数，不超过每日上限则奖励用户对应次数。

## 安全要求

1. **内容审核**：在 `createFeed` 和 `addComment` 中调用微信云开发 `cloud.openapi.security.msgSecCheck` 进行内容安全检测，违规内容拒绝发布
2. **权限校验**：删除操作必须校验操作为本人
3. **仅公开动态**：广场和他人主页只展示公开动态

## 开发规范

- 统一使用 `wx-server-sdk`
- 使用 `async/await` + `try/catch` 错误处理
- 返回格式统一：`{success: true, data: ...}` 或 `{success: false, error: ...}`
- 遵循现有项目代码风格，保持一致
