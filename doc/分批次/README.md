# 去玩匹配 - 动态功能分批次开发提示词

## 批次说明

由于一次性开发 8 个云函数 + 4 个页面代码量较大，AI输出容易截断，所以分两个批次开发：

| 批次 | 内容 | 文件 |
|------|------|------|
| 批次1 | 云函数开发（8个） | [批次1-数据库和云函数.md](./批次1-数据库和云函数.md) |
| 批次2 | 前端页面开发（4个新页面 + 修改现有文件） | [批次2-前端页面.md](./批次2-前端页面.md) |

## 使用方法

按顺序喂给 AI：

1. 先发送 `批次1-数据库和云函数.md`，等待 AI 输出完所有云函数代码
2. 然后发送 `批次2-前端页面.md`，等待 AI 输出完所有前端页面代码
3. 最后将代码复制到项目对应位置，上传部署云函数即可
