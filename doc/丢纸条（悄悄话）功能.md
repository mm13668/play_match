帮我在「去玩匹配」小程序中新增"丢纸条"功能，需求如下：
# 1. 需求逻辑
- 个人中心，用户必须有昵称和性别才能丢纸条，信息不全不允许丢
- 纸条只能丢给**最近7天活跃过的一位随机异性**，一对一投递
- 用户发纸条前，需要填写纸条内容，消耗一个贝壳才能发送
- 用户收到纸条后，需要消耗一个贝壳才能打开，打开后查看纸条发送者的性别昵称和纸条内容
- 打开后可以选择是否生成双方的姓名情侣匹配报告（复用现有情侣匹配功能）
- 每个纸条只能回复一次，发送者收到回复可以查看对方信息
- 如果想进一步联系，只能通过回复纸条内容交换联系方式
# 2. 数据库设计
- 新增集合 notes，字段：
  - _id: 纸条ID
  - sender_openid: 发送者openid
  - sender_nick: 发送者昵称
  - sender_gender: 发送者性别
  - content: 纸条内容
  - require_gender: 接收性别要求（固定为异性）
  - receiver_openid: 接收者openid
  - create_time: 创建时间
  - is_viewed: 是否已被接收者查看
  - is_replied: 是否已回复
  - reply_content: 回复内容
  - reply_openid: 回复者openid
  - reply_time: 回复时间
  - match_report_generated: 是否已生成匹配报告
- 在用户表user增加用户最近活跃时间

# 3. 贝壳次数计算
- 剩余次数 = 贝壳个数，计算方式：从user表查出 (ad_free_time + 3) - 累计消耗总数，累计消耗 = 已发送纸条数 + 已查看纸条数
- 每个用户初始赠送3个贝壳
- 没有贝壳了，弹出提示："观看广告可以获得贝壳，继续玩哦"，看完广告增加1个贝壳
# 4. 页面开发
- 在首页**新增一个"丢纸条"卡片入口**
- 在个人中心**增加"丢纸条"入口**，同时在小程序菜单增加"丢纸条"菜单项
- 丢纸条页面：输入纸条内容（最多200字），点击"丢出去"按钮
- 个人中心增加"我的纸条"页面：分"我发出的"和"我收到的"两个标签页
- 列表展示：纸条摘要、发送时间、是否已读/已回复
- 点击收到纸条：
  - 未解锁→显示"观看广告解锁"按钮
  - 已解锁→展示纸条内容+发送者信息+匹配报告入口+回复按钮（仅未回复时显示）
# 5. 核心逻辑
- 丢纸条判断：如果当前用户没填昵称或性别，提示"请先到个人中心完善昵称和性别信息才能丢纸条"
- 系统从所有满足条件（有性别、**最近7天登录过**）的异性用户中随机选一位，分配给该纸条
- 如果当前没有满足条件的异性，提示"暂无符合条件的异性，稍等再来试试吧"，不消耗贝壳
- 用户点击查看→扣除一个贝壳→触发激励视频广告→广告看完标记为已查看→展示内容
- 回复内容限制最多200字，回复按钮：只有未回复的纸条可以回复，回复一次后不能再回复
- 点击"生成情侣匹配报告"→自动填入双方昵称，选择情侣默契测试，跳转到匹配报告生成流程
# 6. 变现规则
- 发送纸条：消耗1贝壳，无贝壳需看广告得贝壳才能发送
- 查看纸条：消耗1贝壳，无贝壳需看广告得贝壳才能查看
- 重复查看同一个纸条不扣贝壳，每个纸条只需要消耗一次
- 每看一次广告奖励1个贝壳
# 7. 样式设计
- 卡片式设计，简洁干净，符合匿名社交的神秘感
- 文字配色柔和，突出神秘感和期待感
# 8. 内容安全
- 提交纸条和回复时，需要调用微信内容安全接口审核文本，违规内容不允许发送，不消耗贝壳
